Hur man kan använda Github i Skolan

Github är ett så kallat "Socialt nätverk för utvecklare", där folk kan hjälpa till att jobba på olika Open Source projekt.

Här står det hur man kan använda det för uppgifter i skolan, ungefär som Edwise.



Hur man använder det som Elev

Steg 1 - Skaffa ett Github Konto

Gå in på https://github.com, och använd formuläret på framsidan.

OBS! Se till att använda en fungerande email-adress.

Steg 2 - Kopiera grundarkivet[1] från Gruppen PRRPRR01TE2 till din egen användare

Gå in på https://github.com/PRRPRR01TE2/uppgifter.
Tryck på "Fork" [2] uppe i det högra hörnet.

Ett eget arkiv, med samma namn, kommer nu att skapas på din användare.

Steg 3 - Ladda ner & installera Github-klienten

Gå in på http://windows.github.com och ladda ner applikationen.

Starta GitHubSetup.exe, låt den ladda ner och installera applikationen.

När installationen är klar kommer programmet upp och ber dig logga in. Logga då in med den informationen du Skapade i Steg 1.

Under "configure git", fyll i ditt namn och din email-adress som du använde när du skapade kontot på Github.

Skippa steget "repositories".

Steg 4 - Ladda ner det kopierade arkivet och konfigurera din IDE[3] till samma plats

I Github-klienten, klicka på ditt användarnamn under "github". Du borde då se ditt arkiv "uppgifter" komma upp med namnet *användarnamn*/uppgifter.

Högerklicka på arkivet och tryck på "clone to..."

Välj en basmapp där du vill lägga arkivet.

När den har laddat ner klart så dubbelklickar du på arkivet. Du får då fram all uppdaterings-historik för arkivet. Det borde bilden nedan.

Lite info

När du har "klonat" ner arkivet kommer du hitta det under *basmapp*\uppgifter
Alltså, om du skulle välja mappen C:\Users\ArmedGuy\Documents\GitHub, så skulle du hitta arkivet under C:\Users\ArmedGuy\Documents\GitHub\uppgifter.

Detta kommer att bli det stället där du kommer att lägga all din kod som ska laddas upp.

Nu ska du ändra din "workspace" i Eclipse/NetBeans. Denna guide visar endast för Eclipse.

Starta Eclipse som vanligt(ifall att "Select workspace" rutan kommer upp, ta det som du vanligtvis tar.

I Eclipse, öppna File -> Switch Workspace -> Other. Spara det gamla värdet, sedan trycker du "Browse" och söker upp *basmapp*\uppgifter som du valde tidigare.

Eclipse kommer att starta om, och du kommer att ha en tom vy.

För att importera alla projekt från den gamla arbetsytan, klicka på File -> Import och välj General -> Existing Projects into Workspace

I "Select root directory", bläddra till den gamla arbetsytan, och bocka för de projekt du vill flytta.

OBS! Se till att "Copy projects into workspace" är förbockad.

Nu är allt klart, och du kan börja på uppgifterna!

Steg 5 - Hur man laddar upp något efter att man är klar

I Github-klienten, klicka på ditt användarnamn under "github". Du borde då se ditt arkiv "uppgifter" komma upp med namnet *användarnamn*/uppgifter.

Dubbelklicka på arkivet för att komma in i historievyn.

Om du har gjort ändringar av filer i ditt arkiv borde rutan se ut ungefär såhär(notera "uncommited changes" till vänster)

För att skicka ändringarna till Github, skriv in en kort beskrivning under "Commit Message". Om du måste skriva något mer kan du använda "Extended description".

Tryck sedan på "Commit". I detta steg har ändringarna låsts och fått ett s.k. "Commit ID", men ännu inte laddats upp servern.(notera "unsynced commits").

För att slutligen ladda upp ändringarna, tryck på "sync" och vänta på att den laddar upp. När den är klar får du en länk till GitHub där du kan se över ändringarna.





Beskrivningar

Arkiv

Arkiv, på engelska "Repository", är där din kod ska sparas. Arkivet innehåller en historik av varje uppdatering("commit", läs steg 5 för mer info) samt hur hela arkivet såg ut vid det tillfället.

Vid grupprojekt kan flera stycken arbeta på ett arkiv(om man är vidare intresserad av revision/source control, läs http://en.wikipedia.org/wiki/Revision_control).

Fork

Inte för att misstas för en vanlig gaffel. Att "Forka"(svengelska) betyder att "dela upp" eller att "gå skilda vägar". På github betyder det att man kopierar någon annans projekt, för att "gå sin egen väg" med det.

IDE

IDE står för "Integrated development environment" och menar ett avancerat utvecklingsverktyg för programmering.
